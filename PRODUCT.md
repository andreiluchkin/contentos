# ContentOS — Content Production Operating System

> Версия: 2.0 · Дата: 2026-06-17 · Автор: Andrei Luchkin

---

## Что это

ContentOS — персональная операционная система для производства контента.

Не просто планировщик постов. Это система которая берёт сырую идею из любого источника и за 3 минуты превращает её в качественный контент для нужной платформы — в голосе автора, по правилам платформы, с измеримым качеством.

Один инструмент закрывает весь производственный цикл: идея → черновик → оценка качества → публикация.

Личный инструмент одного автора. Не SaaS.

---

## Проблема

Регулярный постинг на нескольких платформах — это производственный процесс. Без системы он разваливается:

- Идеи рождаются и теряются — нет единого места их собрать
- Одна идея = ручной рерайт для каждой платформы
- Нет понимания в каком статусе каждая единица контента
- Публикация требует присутствия в реальном времени
- Контент получается хаотичным — нет баланса тем
- AI генерирует типовой текст, не голос автора

---

## Решение

Контент-конвейер с полным контролем качества:

1. Идеи приходят автоматически из внешних источников (Reddit-проект и другие)
2. Batch Mode: одобряю пачку → система генерирует 10 черновиков сразу
3. Content Score оценивает качество до публикации — объясняет что слабо
4. Каждый пост создаётся по шаблону своей платформы, не просто "переписывается"
5. Тип контента (кейс, инструкция, мнение...) определяет структуру генерации
6. Brand Voice и Knowledge Base делают контент узнаваемым и уникальным
7. Repurpose Engine: одна мысль → десятки единиц контента
8. Content Pillars: баланс тем виден в календаре

## Content Pipeline — статусы контента

Каждая единица контента проходит явный путь. Без статусов при 100+ постах в месяц — хаос.

```
Inbox          → новая идея из внешнего источника (Reddit-проект и др.)
Idea Approved  → я одобрил идею, она готова к генерации
Draft          → AI сгенерировал черновик (или я написал вручную)
Review         → черновик на моей проверке + Content Score рассчитан
Ready          → пост одобрен, ждёт назначения времени
Scheduled      → время назначено, пост в очереди на автопубликацию
Published      → опубликован успешно
Error          → ошибка публикации, требует внимания
```

Переходы между статусами:
- `Inbox → Idea Approved` — моё действие (одобрить / отклонить)
- `Idea Approved → Draft` — автоматически или по кнопке "Создать черновик"
- `Draft → Review` — автоматически после генерации
- `Review → Ready` — моё действие после проверки
- `Ready → Scheduled` — я назначаю время
- `Scheduled → Published` — автоматически в назначенное время

## JTBD — Jobs To Be Done

### J1 — Идея-конвейер: внешний источник → Inbox

**Когда:** Хочу писать актуально и не хочу думать о темах  
**Хочу:** Получать готовые идеи из Reddit-проекта прямо в ContentOS  
**Чтобы:** Никогда не было ступора "о чём писать сегодня"

**Сценарий:**
1. Reddit-проект анализирует тренды → пушит идею в ContentOS через API: заголовок, контекст, источник, предлагаемая платформа, оценка релевантности
2. Идея падает в Inbox со статусом `Inbox`
3. Утром я вижу 5–10 идей с превью
4. Одобряю нужные → статус `Idea Approved`
5. Отклоняю лишние → система запоминает паттерн

**Batch Mode:** выбираю 10 идей → "Создать черновики" → система генерирует все 10 сразу. Потом только редактирую.

### J2 — Создание поста: тип контента + платформа + шаблон

**Когда:** Есть идея, хочу превратить её в пост  
**Хочу:** Выбрать тип контента и платформу — получить структурированный черновик  
**Чтобы:** Не писать с нуля и не получать типовой AI-текст

**Первый выбор — тип контента** (определяет структуру генерации):

| Тип | Структура |
|-----|-----------|
| Кейс | Ситуация → Проблема → Решение → Результат |
| Разбор | Что анализируем → Ключевые выводы → Инсайт |
| Инструкция | Зачем → Шаги → Результат |
| Мнение | Тезис → Аргументы → Вывод |
| Подборка | Введение → N пунктов → Итог |
| История | Контекст → Конфликт → Развязка → Урок |
| Наблюдение | Что заметил → Почему важно → Что делать |
| Ошибка | Что сделал → Что пошло не так → Что понял |
| Урок | Что узнал → Как применить → Пример |
| Запуск продукта | Что, Кому, Почему сейчас → CTA |

**Второй выбор — платформа** (определяет формат и шаблон):

**Telegram:**
```
→ Сильный первый абзац (читатель решает читать ли дальше)
→ Короткие блоки по 2–4 строки, пробелы между блоками
→ Личное мнение или позиция автора
→ Вывод в последнем абзаце
→ Без хэштегов
```

**Instagram:**
```
→ Hook: первые 125 символов видны без "Ещё" — должны цеплять
→ История или инсайт в теле
→ Чёткий инсайт или ценность
→ CTA в конце (вопрос или призыв)
→ 5–10 хэштегов отдельным блоком
```

**LinkedIn:**
```
→ Конфликт или провокация в первой строке
→ Личный опыт как доказательство
→ Урок или вывод
→ Чёткий CTA
→ Профессиональный, но живой тон
```

**TikTok (только скрипт):**
```
→ Hook (первые 3 сек): вопрос или неожиданное утверждение
→ Удержание: развитие идеи с нарастанием
→ Payoff: главный инсайт / ответ
→ CTA: подписка / комментарий / следующий ролик
```

**X (тред):**
```
→ Твит 1: тезис + обещание (что узнает читатель)
→ Твиты 2–N: один аргумент = один твит
→ Финал: вывод + CTA
```

**Два режима публикации:**
- *Одна платформа* — создаю пост под конкретный канал
- *Мультиплатформа* — тема одна, версии для каждой выбранной платформы, редактирую независимо

### J3 — Content Score: качество измеримо

**Когда:** Черновик готов, перехожу в статус Review  
**Хочу:** Понять насколько хорош пост до публикации  
**Чтобы:** Не публиковать слабый контент

**Оценка автоматически рассчитывается при переходе Draft → Review:**

```
Content Score: 84 / 100

Hook          92  ████████████████████░░░  Сильное начало
Structure     88  █████████████████████░░  Чёткая структура
Readability   81  ████████████████████░░░  Легко читается
CTA           60  ███████████████░░░░░░░░  ⚠ Нет призыва к действию
Platform Fit  98  ██████████████████████░  Идеально под платформу
```

**AI объясняет конкретно:**
- "Слабый первый абзац — читатель не поймёт зачем читать дальше"
- "Нет CTA — непонятно что делать после прочтения"
- "Слишком длинное вступление — убери первые два предложения"
- "Третий пункт — вода, не несёт ценности"

**Кнопка "Улучшить"** — AI применяет правки и пересчитывает Score.

### J4 — Repurpose Engine: одна мысль → много контента

**Когда:** Есть голосовая заметка, видео или длинный материал  
**Хочу:** Получить из него контент для всех платформ без ручной работы  
**Чтобы:** Одно усилие = недели контента

**Голосовая заметка → контент:**
```
Голосовая заметка (2–5 мин)
  → Telegram пост (тип: Мнение или Наблюдение)
  → LinkedIn пост (тип: Урок или Кейс)
  → Тред X (разбивка на тезисы)
  → Сценарий Reels / Shorts (хук + тело + CTA)
```

**YouTube видео → контент:**
```
YouTube видео (10–30 мин)
  → Транскрипция (Whisper)
  → 5–10 постов (каждый ключевой тезис = пост)
  → 3–5 сценариев Reels (сильные отрезки 30–60 сек)
  → 5 твитов или 1 тред
  → 1 статья (длинный Telegram-пост или LinkedIn-статья)
```

**Результат:** ContentOS раскидывает единицы контента по Content Backlog с разными датами — не перегружает один день.

### J5 — Утренний 10-минутный ритуал

**Когда:** Начинаю рабочий день  
**Хочу:** За 10 минут закрыть всё по контенту  
**Чтобы:** Не возвращаться к этому в течение дня

**Сценарий:**
1. Открываю ContentOS — вижу: Inbox (N новых идей), что выходит сегодня, где дыры на неделе
2. Batch Mode: выбираю 5–10 идей → "Создать черновики" → жду 30–60 сек
3. Просматриваю черновики, корректирую слабые места (Content Score показывает где)
4. Назначаю время → пост уходит в `Scheduled`
5. Всё. День закрыт.

**Умное расписание:**
- Знает оптимальное время для каждой платформы (настраивается)
- Предлагает ближайший свободный слот при назначении
- Не ставит два поста в одну соцсеть в один день без явного разрешения

### J6 — Brand Voice + Knowledge Base: контент звучит как я

**Когда:** Хочу чтобы AI писал в моём стиле, а не типовым языком  
**Хочу:** Загрузить свой голос и знания один раз — использовать в каждой генерации  
**Чтобы:** Контент был узнаваемым и уникальным

**Brand Voice хранит:**
- Стиль письма: прямой, без воды, с позицией
- Любимые конструкции и обороты
- Запрещённые слова и фразы ("синергия", "уникальный", "революционный")
- Длина текстов по платформам
- 5–10 примеров лучших постов (эталон голоса)

**Knowledge Base хранит:**
- Личные кейсы и опыт
- Заметки и наблюдения
- Мысли которые ещё не стали постами
- Старые посты (источник для репёрпоза)
- Данные о своих проектах

**При генерации:** AI сначала читает Brand Voice → проверяет Knowledge Base на релевантный опыт → генерирует с учётом обоих.

## Разделы продукта

### 1. Главная — Inbox + дашборд дня

Единственная страница которую открываю утром.

**Что здесь:**
- **Inbox** — входящие идеи от Reddit-проекта и других источников. Карточка: тема, источник, контекст, предлагаемая платформа, оценка релевантности. Кнопки: Одобрить / Отклонить / Редактировать.
- **Batch Mode** — выбрать несколько идей → "Создать черновики"
- **Сегодня** — что запланировано, по каким платформам, в какое время
- **Дыры недели** — визуальная сетка Пн–Вс, пустые слоты выделены

**Цель:** Открыл → 10 минут → закрыл. Весь контент-план готов.

---

### 2. Content Backlog

Главный рабочий экран автора. Полный список всего контента — от идей до опубликованных.

**Что здесь:**
- Список всех единиц контента со статусами из Content Pipeline
- **Фильтры:** по статусу, по пиллару, по платформе, по типу контента
- **Быстрые действия:** создать черновик, назначить время, открыть на редактирование
- Счётчики по пиллару: сколько единиц в каждом, баланс

**Почему отдельный экран, не календарь:** Календарь показывает время. Backlog показывает производственный процесс. Это разные задачи.

---

### 3. Создать пост / Редактор

**Флоу создания:**
```
Выбрать тип контента (Кейс / Мнение / Инструкция / ...)
    ↓
Выбрать платформу(ы)
    ↓
Ввести тему или вставить черновик
    ↓
AI генерирует по шаблону платформы + Brand Voice + Knowledge Base
    ↓
Content Score рассчитывается
    ↓
Редактирую / нажимаю "Улучшить"
    ↓
Назначаю время → Scheduled
```

**Для мультиплатформы:** вкладка на каждую платформу, редактирую независимо, свой Content Score для каждой версии.

---

### 4. Repurpose Engine

Отдельный инструмент. Не часть обычного редактора.

**Входы:**
- Голосовая заметка (запись прямо в браузере или загрузка файла)
- Видеофайл или YouTube-ссылка
- Длинный текст

**Что происходит:**
1. Транскрипция (для аудио/видео)
2. AI выделяет ключевые мысли
3. Для каждой мысли — предложения по типу контента и платформе
4. Одним кликом отправляю в Backlog
5. ContentOS предлагает распределение по датам (не перегружает один день)

---

### 5. Календарь публикаций

Обзор того что выходит по времени. Не рабочий экран — экран планирования.

**Что здесь:**
- Месячный вид (переключение на неделю)
- В ячейке дня: иконки платформ + количество постов + цвет пиллара
- Клик на день → список постов с временем и статусом
- Фильтр по платформе и по пиллару
- Статусы: запланирован (серый) / опубликован (зелёный) / ошибка (красный)

**Распределение по пилларам видно в цвете** — сразу понятно нет ли перекоса в темах.

---

### 6. Content Pillars

Сущность которая структурирует о чём я пишу.

**Что такое пиллар:**
Тематическая рубрика. Например: AI, Продакт-менеджмент, Личный бренд, Стартапы, Маркетинг, Разборы продуктов.

**Как работает:**
- Каждый пост принадлежит пиллару
- В Backlog вижу сколько единиц в каждом пилларе
- В Календаре каждый пиллар — свой цвет
- Система предупреждает если один пиллар занимает >50% контента за неделю

---

### 7. Brand Voice & Knowledge Base

Настройки один раз — работают везде.

**Brand Voice:**
- Примеры лучших постов (загружаю 5–10 эталонных текстов)
- Стиль: тон, длина, структурные предпочтения
- Запрещённые слова и клише
- Длина по платформам

**Knowledge Base:**
- Заметки и мысли (быстрый ввод с главного экрана)
- Загрузка документов, старых постов
- Поиск по базе при генерации
- AI ищет релевантный личный опыт и вставляет в пост

---

### 8. Интеграции

**Подключённые соцсети:**
- OAuth для каждой платформы
- Статус токенов (активен / истекает / ошибка)
- Автообновление токенов

**Входящие источники:**
- API-эндпоинт `/api/inbox/idea` для вебхуков от Reddit-проекта и других
- Список авторизованных источников с API-ключами
- Лог входящих идей

**Обратная связь в Reddit-проект:**
- Раз в неделю ContentOS отправляет сигнал: какие темы работают, какие нет
- Reddit-проект использует это для приоритизации

---

## Что не делаем сейчас

Намеренно исключено из приоритетов:

- Сложная аналитика и детальные отчёты по платформам
- Drag & drop в календаре
- Медиа-библиотека (используем простую загрузку файлов)
- Сложные интеграции с X (Twitter API дорогой)
- Stories для Instagram
- A/B тесты контента

Причина: это не влияет на качество производства контента. Главное — скорость и качество от идеи до публикации.
