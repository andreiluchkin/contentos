# Разграничение: design/ vs ui-ux-pro-max

Два инструмента работают в одном проекте и не должны конфликтовать.
Каждый отвечает за свою область — ни один не перезаписывает решения другого.

---

## Что за что отвечает

### design/ — Бренд ContentOS (источник истины)

**Владелец:** Andrei Luchkin (файлы залиты вручную)

**Что хранится:**
- `DESIGN.md` — описание визуального языка проекта (Jitter-стиль)
- `tokens.json` — конкретные значения токенов бренда
- `variables.css` — CSS-переменные под этот проект
- `theme.css` — тема

**Правила:**
- Это **неизменяемая база**. Менять только вручную или по явному запросу владельца.
- Отсюда берутся: цвета (`#7a40ed`, `#19171c`, `#f2f1f3`), радиусы (50px кнопки, 40px карточки), шрифты (TWK Lausanne + Inter), тени (многослойные диффузные).
- AI-скилл `ui-ux-pro-max` **не трогает** эти файлы.

---

### .claude/skills/ui-ux-pro-max/ — Дизайн-интеллект (инструмент)

**Владелец:** ui-ux-pro-max skill (nextlevelbuilder)

**Что делает:**
- Генерирует структуру компонентов, UX-паттерны, советы по доступности
- Подбирает layout-решения, анимации, навигационные паттерны
- Проверяет соответствие стандартам (44px touch targets, контраст 4.5:1, и т.д.)

**Правила:**
- Работает **на основе токенов из design/**, а не генерирует свои цвета/шрифты
- Когда скилл предлагает цвет или радиус — берём из `design/tokens.json`, не из базы скилла
- Предложения скилла по типографике применяем только если TWK Lausanne недоступен (fallback: Inter Tight, Switzer)
- Скилл **не создаёт** файлы в `design/`

## Протокол взаимодействия

```
Запрос на UI-задачу
        │
        ▼
1. Читаем design/tokens.json + design/DESIGN.md  ← бренд-контекст
        │
        ▼
2. Запускаем ui-ux-pro-max для структуры и UX    ← дизайн-интеллект
        │
        ▼
3. Применяем токены бренда к решению скилла      ← синтез
        │
        ▼
4. Пишем код компонента
```

## Конфликт токенов — как решать

| Параметр | Источник истины |
|----------|-----------------|
| Цвета | `design/tokens.json` — всегда |
| Радиусы | `design/tokens.json` — всегда |
| Тени | `design/tokens.json` — всегда |
| Шрифты | `design/DESIGN.md` — TWK Lausanne / Inter |
| Spacing | `design/tokens.json` (4px base) |
| Touch targets | ui-ux-pro-max (мин. 44×44px) |
| Доступность | ui-ux-pro-max (контраст, aria) |
| UX-паттерны | ui-ux-pro-max (навигация, формы) |
| Анимации | ui-ux-pro-max (150–300ms), цвета из design/ |

**Правило большого пальца:** если токен есть в `design/` — берём оттуда. Если нет — доверяем ui-ux-pro-max.

---

## Что никогда не делать

- ❌ Не генерировать новую палитру через ui-ux-pro-max если проект ContentOS
- ❌ Не менять `design/*.css` и `design/tokens.json` автоматически
- ❌ Не использовать дефолтные цвета ui-ux-pro-max (синие/зелёные из его базы) вместо фиолетового бренда
- ❌ Не устанавливать острые углы (radius < 20px) — это нарушает бренд
- ❌ Не использовать emoji как иконки (запрет обоих инструментов)
